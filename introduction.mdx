HCEGuard is a Windows desktop application that brings secure, rolling-code authentication to Halo CE multiplayer servers. Players register once and then sign in with a nickname and a 6-digit one-time password (OTP) that changes with every session — so unauthorized players cannot reuse stolen credentials or impersonate others. Server administrators get reliable identity control without requiring complex infrastructure on the player's side.

## Who HCEGuard is for

HCEGuard is for **players** who want to join protected Halo CE servers and for **server administrators** who want verifiable player identities. If you play on a server that requires HCEGuard, you need to create an account and keep the app running while you play. If you run a server, HCEGuard's integration layer handles validation automatically once players are authenticated.

## How authentication works

HCEGuard uses a server-side account linked to your nickname. When you sign in, the app sends your nickname and OTP to the HCEGuard service for verification. On success, the app receives a session token and activates your local session. If Halo CE integration is enabled, HCEGuard immediately syncs the validation files your game client needs before you launch Halo CE.

After three consecutive failed login attempts the app displays a CAPTCHA challenge to prevent automated access attempts. Once you complete the CAPTCHA and sign in successfully, the counter resets.

<Note>
  HCEGuard only runs on Windows. It does not modify game files, inject code into running processes, or install system drivers.
</Note>
