HCEGuard uses one-time passwords (OTPs) to verify your identity every time you sign in. Instead of a fixed password that stays the same, an OTP is a short numeric code that changes constantly — each code is only valid for a brief window of time. This makes your credentials significantly harder to steal or share, keeping authenticated servers more secure for everyone.

## What is an OTP?

An OTP is a single-use, time-based verification code generated by your authentication provider. Because it expires quickly, intercepting or guessing a code after the fact does not give an attacker access to your account. You receive a fresh code each time you need to sign in.

## Why HCEGuard uses OTP

Traditional passwords are static — once leaked, they remain compromised until changed. HCEGuard replaces static passwords with OTPs to address two specific problems in online gaming:

- **Credential sharing** — a constantly rotating code cannot be handed to someone else for reuse. Each code is tied to a specific moment in time.
- **Replay attacks** — a code intercepted during one login attempt cannot be used again in a later attempt, because it will already be expired.

This approach protects both your account and the integrity of the servers you play on.

## How to enter your OTP

1. Open HCEGuard and type your nickname in the **Nick** field.
2. Retrieve your current OTP from your authentication provider.
3. Type the 6-digit code into the **OTP Code** field. The field only accepts numeric digits — letters and symbols are blocked automatically.
4. Click **Enter** (or press Enter) to sign in.

Keep the following in mind when entering your OTP:

- The code must be **exactly 6 digits**. Shorter or longer entries will be rejected before the login attempt is sent.
- **Numbers only** — the field does not accept letters or special characters.
- OTP codes expire quickly. If a code is rejected, retrieve a fresh one from your authentication provider and try again immediately.

## If your OTP is rejected

A rejected OTP usually means one of the following:

- **The code expired** — OTPs are time-sensitive. Retrieve a new code from your authentication provider and enter it without delay.
- **A digit was misread** — double-check that all 6 digits are entered correctly before retrying.
- **Your account is under review or suspended** — if HCEGuard shows a message about your account status rather than an "invalid credentials" error, the OTP itself is not the problem. See [Account registration](/account/registration) for more on account review.

## CAPTCHA after repeated failures

If you enter incorrect credentials three times in a row, HCEGuard displays a visual CAPTCHA challenge before allowing another attempt. The CAPTCHA shows a 5-character alphanumeric code rendered in a distorted image.

To continue:

1. Read the characters displayed in the CAPTCHA image.
2. Type them into the CAPTCHA field. The check is not case-sensitive.
3. If the image is difficult to read, click the refresh button (↺) next to the image to generate a new one.
4. Submit your login credentials along with the CAPTCHA answer.

The CAPTCHA remains active until you sign in successfully. Solving it does not reset your failed attempt count — the next successful login does.
